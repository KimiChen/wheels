# Changelog

## 0.2.0 - 2026-07-11

- 移除对 `wedecode` 命令和 npm 包的运行时依赖，改为项目内置离线反编译引擎。
- 基于 `wedecode v0.10.6` 移植小程序、小游戏、分包、WXML、WXSS、WXS 和 JavaScript 恢复流程。
- 新增 V1MMWX 加密包的项目内解密，以及严格的 wxapkg 头部、索引、数据范围和 UTF-8 校验。
- 新增路径穿越、符号链接、硬链接、Windows 路径别名、大小写冲突和危险 `--clear` 保护。
- 改为由父进程安全解包，包内代码只在无文件写入、网络、子进程和嵌套 worker 权限的只读进程中分析。
- 新增内存文件 overlay 和敌对 manifest 校验，由父进程限制路径、字段、操作数、文件大小和总写入量后落盘。
- 新增 worker 完成消息关联与进程控制入口收紧，拒绝伪造响应、直接写文件、外部符号链接和进程信号能力。
- VM 语法错误和执行超时不再被吞掉；CSS 片段也统一通过带超时的 VM 执行。
- 将运行时要求调整为 Node.js 25+，强制使用包含网络限制的 Node 权限模型；旧版本不再以降级隔离运行。
- 主包改为按包内容识别并优先处理；目录反解默认排除 `publicLib.wxapkg`。
- 完整反编译统一将 `__plugin__/` 映射为 `plugin_/`，插件枚举同时读取磁盘文件与内存 overlay。
- 输入包检查记录解密后 SHA-256，拒绝检查和解包之间发生内容变化的缓存文件。
- 生成的 WXSS、WXS、WXML 和插件引用路径统一使用 POSIX `/` 分隔符。
- 自定义 polyfill 先复制到私有只读快照，worker 不再授权可变的源目录；快照和读取授权路径均在 `--clear` 前完成校验。
- `--dry-run` 改为打印内置引擎执行计划和工作目录，删除 `--wedecode-bin`。
- 新增 npm 发布白名单、提交前自动检查、开发文档、Babel MIT 许可说明和 CLI 版本单一来源。
- 扩充安全解包、加解密、CLI、VM、worker、manifest 和真实流程测试；授权四包样例两次生成的 139 个文件逐文件 SHA-256 一致。

## 0.1.0

- 新增微信小游戏/小程序 `wxapkg` 缓存扫描。
- 新增候选目录按修改时间倒序展示，显示游戏名称、标识 ID、包数量、大小和包清单。
- 新增离线身份线索识别：从 appid 目录附近的 metadata 文件读取游戏名称，避免把缓存路径段误当名称。
- 新增 `--appid`、`--since`、`--limit`、`--list-packages`、`--dry-run` 等常用参数。
- 新增输出目录存在时的确认流程和 `--clear` 风险提示。
- 新增 `wedecode` 调用封装、默认输出目录生成和反解后输出统计。
- 新增反解失败排查建议。
- 新增 Node.js 单元测试和 `npm run check`。
